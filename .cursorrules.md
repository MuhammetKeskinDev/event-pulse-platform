# EventPulse Project Rules & Standards

## Tech Stack
- Backend: Node.js with TypeScript and Fastify.
- Database: PostgreSQL (TimescaleDB) and Redis.
- Queue: Redis Streams.
- Testing: Jest or Vitest.

## Coding Standards
- Use Functional Programming patterns where possible.
- Strict TypeScript: No 'any', use interfaces/types for all data structures.
- Async/Await: Always use try/catch blocks for async operations.
- [cite_start]Validation: Use Zod or TypeBox for runtime schema validation[cite: 83].
- [cite_start]Logging: Use structured logging (pino) for all service activities[cite: 63].

## Architecture Principles
- Layered Architecture: API -> Service -> Repository/Worker.
- [cite_start]Event-Driven: API must acknowledge immediately (202 Accepted) and process asynchronously[cite: 85, 86].
- [cite_start]Resilience: Implement retry logic for queue consumers (max 3 retries)[cite: 88].
